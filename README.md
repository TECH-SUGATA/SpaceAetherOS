## вңҰ Overview

**AetherOS** is a full-stack SaaS space-tech command platform built with a futuristic glassmorphism UI. It delivers live orbital data, AI-powered space assistance, SpaceX launch tracking, NASA imagery, and near-earth object monitoring вҖ” all in real time.

> *"All systems nominal. Orbital trajectory stable."*

## вңҰ Getting Started

### Prerequisites

- Node.js 18 or higher
- npm 9 or higher
- MongoDB Atlas account (free)
- NASA API key (free)
- Google Gemini API key (free)

### Local Development

```bash
# 1. Clone the repository
git clone https://github.com/TECH-SUGATA/SpaceAetherOS.git
cd SpaceAetherOS

# 2. Install backend dependencies
cd backend
npm install

# 3. Configure environment variables
cp .env.example .env
# Edit .env with your API keys

# 4. Start the backend server
npm run dev
# Server starts at http://localhost:5000

# 5. Open the frontend
# Open frontend/aether_os.html in your browser
# Or use VS Code Live Server for best experience
```

## вңҰ Environment Variables

Copy `backend/.env.example` вҶ’ `backend/.env` and fill in all values:

```env
# Server
PORT=5000
NODE_ENV=development

# Database
MONGODB_URI=mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/aetheros

# Authentication
JWT_SECRET=your_super_secret_jwt_key_min_32_chars

# NASA API вҖ” get free key at api.nasa.gov
NASA_API_KEY=DEMO_KEY

# Google Gemini AI вҖ” get free key at aistudio.google.com
GEMINI_API_KEY=AIzaSy...

# Firebase (optional вҖ” for Google login)
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

# Frontend URL (for CORS)
FRONTEND_URL=https://space-aether-os.vercel.app
```

| Variable | Required | Description |
|---|---|---|
| `MONGODB_URI` | вң… Yes | MongoDB Atlas connection string |
| `JWT_SECRET` | вң… Yes | Minimum 32 character random string |
| `NASA_API_KEY` | вң… Yes | Use `DEMO_KEY` for testing |
| `GEMINI_API_KEY` | вӯҗ Recommended | AI Oracle chatbot responses |
| `FIREBASE_*` | рҹ”ө Optional | Required only for Google OAuth login |

## вңҰ Socket.io Events

```javascript
const socket = io('https://spaceaetheros.onrender.com');

// Live ISS Position (updates every 5 seconds)
socket.emit('subscribe:iss');
socket.on('iss:update', ({ latitude, longitude, altitude, velocity }) => {
  console.log(`ISS вҶ’ Lat: ${latitude}, Lon: ${longitude}`);
});

// SpaceX Launch Updates
socket.emit('subscribe:launches');
socket.on('launch:update', (launchData) => {
  console.log(launchData);
});

// User Notifications (requires auth)
socket.emit('subscribe:notifications', { userId: 'USER_ID' });
socket.on('notification:new', (notification) => {
  console.log(notification.message);
});
```

## вңҰ Deployment Guide

### Backend вҶ’ Render.com

1. Go to [render.com](https://render.com) вҶ’ **New** вҶ’ **Web Service**
2. Connect your `TECH-SUGATA/SpaceAetherOS` GitHub repository
3. Configure the service:

| Setting | Value |
|---|---|
| **Name** | aetheros-backend |
| **Region** | Singapore |
| **Runtime** | Node |
| **Build Command** | `cd backend && npm install` |
| **Start Command** | `cd backend && npm start` |
| **Plan** | Free |

4. Add all environment variables from the table above
5. Click **Deploy Web Service** вң…

### Frontend вҶ’ Vercel

1. Go to [vercel.com](https://vercel.com) вҶ’ **New Project**
2. Import `TECH-SUGATA/SpaceAetherOS` from GitHub
3. Set **Root Directory** to `frontend`
4. Set **Framework Preset** to `Other`
5. Click **Deploy** вң…

### Database вҶ’ MongoDB Atlas

1. Go to [mongodb.com/cloud/atlas](https://mongodb.com/cloud/atlas)
2. Create free **M0 cluster** (512MB storage)
3. Create database user and whitelist `0.0.0.0/0` for Render access
4. Copy connection string вҶ’ add to `MONGODB_URI` environment variable вң…

## вңҰ Security

- All API keys stored as environment variables вҖ” never hardcoded
- JWT tokens with expiration for session management
- bcrypt password hashing (12 salt rounds)
- Helmet.js HTTP security headers
- Express rate limiting to prevent abuse
- CORS configured to allow only the frontend URL
- MongoDB input sanitization via express-validator
